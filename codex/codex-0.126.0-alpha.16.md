## アプリ MCP パスオーバーライドの追加

`[features.apps_mcp_path_override]` 設定オプションが追加され、ビルトインのアプリ MCP パスをオーバーライドできるようになった。既存のホストおよびベース URL の導出ロジックは維持され、カスタムパスはベース URL の後に追加される。

[参考リンク](https://github.com/openai/codex/pull/20231)

## デフォルトポート使用中時のログインコールバックポートフォールバック

Cursor と Codex Desktop が同じログインコールバックポート（1455）を使用する際の競合が解消された。ポート 1455 が使用中の場合、既存リスナーのキャンセルを試みた後、フォールバックポート（1457）が使用されるようになった。

[参考リンク](https://github.com/openai/codex/pull/19334)

## リモートプラグインのアンインストール前に詳細取得を必須化

プラグインのアンインストール時にリモートプラグインの詳細が事前にフェッチされるようになった。これにより、マーケットプレース名前空間とプラグイン名を正確に特定してローカルキャッシュを適切にクリーンアップでき、バックエンドとローカルキャッシュの状態の不整合が防止される。

[参考リンク](https://github.com/openai/codex/pull/19966)
