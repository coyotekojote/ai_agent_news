## Phase 2 メモリのリトライ改善

Phase 2 メモリジョブがリトライ回数の上限に達した後でもグローバルロックを取得できるようになった。これにより、繰り返し失敗した後も未統合のままだったワークスペースのみの変更が統合されるようになった。

[参考リンク](https://github.com/openai/codex/pull/19809)

## クリーンなワークスペースでの不要な書き込み回避

git ベースのメモリワークスペースがすでにクリーンな場合に、不要なデータベース書き込みを回避するよう最適化された。新しい Phase 2 ファイナライザーにより、不要な I/O と競合が削減された。

[参考リンク](https://github.com/openai/codex/pull/19812)
